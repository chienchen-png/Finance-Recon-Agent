# 04-报告生成

## 触发条件

1. 02-数据核对与校验完成后 → 自动按「校验核对报告」模板生成
2. 03-修正执行与填充完成后 → 自动按「处理后报告」模板生成
3. 用户说"报告"/"汇总"/"出报告"，或多个季度全部处理完成 → 按「最终汇总报告」模板生成

## 前置条件检查

- [ ] 🔑 确认身份已初始化：加载 `知识库/用户配置/操作员信息.json`，读取执行人和审核人姓名（V1.3 新增）
- [ ] 确认 Python 环境可运行：`.venv` 虚拟环境存在
- [ ] 确认 `知识库/报告模板库/` 下三个模板文件存在：
  - `校验核对报告_模板.md`
  - `处理后报告_模板.md`
  - `最终汇总报告_模板.md`
- [ ] 确认对应阶段的源数据存在（核对结果 / 修正日志 / 季度报告）

> ⚠ 若 Python 环境或身份不满足，**自动路由至 00-环境自检与初始化 SKILL**。

### 0. 加载操作员信息（所有报告类型共用、优先执行）

> 🔑 V1.3 新增：生成任何报告前必须加载身份信息。

1. 读取 `知识库/用户配置/操作员信息.json`
2. 提取 `operator.name` → 填入模板中所有 `{操作人}` / `{执行人}` 占位符
3. 提取 `reviewer.name` → 填入模板中所有 `{审核人}` 占位符
4. 提取 `operation_code.code` → 填入模板中所有 `{操作编码}` 占位符
5. 若读取失败或必填字段为空 → 暂停报告生成，提示「请先完成身份初始化」

## 核心指令

### 报告类型一：校验核对报告

> **何时生成**：02-数据核对与校验完成后自动触发。
> **模板**：`知识库/报告模板库/校验核对报告_模板.md`

**填充步骤**：

1. 读取核对结果（匹配结果、差异列表、未匹配清单）
2. 读取依据文件元数据（文件名、sheet、行数、列数）
3. 按模板六章结构填充：
   - **一、核对基本信息**：报告编号、日期、期间
   - **二、检查范围**：依据文件 + 目标文件 + 匹配规则
   - **三、核对结果**：总体统计 + A/B/C 差异分级明细
   - **四、未匹配清单**：未匹配的合同编号、人员、客户、金额
   - **五、处理计划**：自动处理项 / 确认后处理项 / 人工处理项 / 未匹配项
   - **六、下一步**：引导用户选择处理策略

4. 保存到 `${project_root}/结果输出/核对结果/核对报告/{QX}核对报告_{日期}.md`

---

### 报告类型二：处理后报告

> **何时生成**：03-修正执行与填充完成后自动触发。
> **模板**：`知识库/报告模板库/处理后报告_模板.md`

**填充步骤**：

1. 读取修正日志（`${project_root}/修改日志/修正记录_*.json`）
2. 读取修正前后的汇总表数据
3. 按模板七章结构填充：
   - **一、处理基本信息**：报告编号、日期、期间、前置核对报告
   - **二、处理范围**：目标文件 + 处理字段
   - **三、处理明细**：A 类执行记录 + B 类确认记录 + C 类人工复核清单 + 跳过项
   - **四、修正前后对比**：汇总表修正前后 + 按季度汇总
   - **五、备份与日志**：备份路径 + 日志路径 + 处理后文件路径
   - **六、遗留问题**：未匹配 / 跳过项 / 搁置项
   - **七、下一步**：引导生成汇总报告或继续下一季度

4. 保存到 `${project_root}/结果输出/核对结果/处理后报告/{QX}处理后报告_{日期}.md`

---

### 报告类型三：最终汇总报告

> **何时生成**：所有季度处理完成 / 用户说"出报告"/"汇总"。
> **模板**：`知识库/报告模板库/最终汇总报告_模板.md`

**填充步骤**：

1. 读取所有季度的核对报告 + 处理后报告
2. 读取所有修正日志（`${project_root}/修改日志/修正记录_*.json`）
3. 读取修正前后的汇总表数据（计算年度差额）
4. 按模板八章结构填充：
   - **一、项目总览**：报告编号、日期、项目周期
   - **二、处理范围总览**：全部表格 + 核对配置
   - **三、逐季度处理结果**：Q1-Q4 各自的核对/修正/未匹配统计
   - **四、全量修正统计**：总体指标 + 按季度趋势 + 按修正类型分布
   - **五、跨季度分析**：高频差异合同 + 跨季度未匹配 + 人员/客户维度汇总
   - **六、结论**：一句话结论 + 主要发现 + 数据质量评价
   - **七、遗留问题与建议**：需处理项 + 改进建议
   - **八、附件清单**：所有季度报告的路径索引

5. 保存到 `${project_root}/结果输出/核对结果/最终汇总报告/回款提成核对报告_{YYYY}年度.md`

## 禁止行为

- 不得在身份未初始化时生成报告（V1.3 新增）
- 不得在无模板时自行编造报告章节结构（必须严格按模板填充）
- 不得在无源数据时生成空报告
- 不得跳过跨季度去重（最终汇总报告中未匹配合同须去重）
- 不得在数据未就绪时提前生成最终汇总报告（需所有季度完成）
