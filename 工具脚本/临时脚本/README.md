# 临时脚本目录

本目录用于存放一次性、项目定制或表格格式定制的 Python 脚本。通用、稳定、跨项目复用的脚本应保留在 `工具脚本/` 根目录。

## 生成规则

- 新增脚本前必须先判断是否可通过现有通用脚本和配置 JSON 解决。
- 仅当需求包含一次性探查、特殊清洗、特殊核对、特殊验证或非常规表格适配时，才在本目录新增脚本。
- 临时脚本必须以 `temp_` 开头，例如 `temp_validate_20260724.py`。
- 不得在脚本名中包含客户名、人员名、合同号、金额等敏感业务标识。
- 临时脚本应优先通过 `--config` 读取配置，不得硬编码原始业务明细。

## 安全边界

临时脚本可以在本地 Python 内部读取 Excel，但输出必须遵守数据隔离规则。

允许输出：sheet 名、字段名、列索引、行列数、差异数量、差异级别统计、必要定位信息、执行摘要。

禁止输出：全量数据行、全量合同清单、全量人员姓名、全量客户名称或编码、全量金额明细、原始单元格 dump、含原始明细的 JSON/CSV/TXT、将 Excel 明细打印到终端。

## 归零行为

系统归零时会清除本目录下的临时脚本和运行产物，保留 `README.md` 和 `.gitkeep`。
