## 📖 这是什么？

这是一个运行在 VS Code 里的 **AI 财务校验助手**。你可以把 Excel 表格丢进指定文件夹，也可以让智能体扫描你指定的系统文件夹；然后用日常语言告诉它"导入数据"、"核对Q1"，它就会自动帮你完成：

- **自动导入**：识别季度分表和汇总表，自动分类存放
- **智能核对**：按合同编号→人员→客户三层匹配，逐条比对金额
- **安全修正**：备份后才修改，A 类差异自动修、B 类逐条确认、C 类人工复核
- **自动报告**：生成专业的三类 Markdown 报告（核对/修正/年度汇总）
- **脱密处理**：客户名称一键替换为编码，敏感列自动删除
- **一键归零**：说"归零"清空所有运行数据，系统退回初始状态，便于测试和复用
- **统一交互确认**：所有需要用户确认的流程都通过 VS Code 的 Ask Questions 组件完成，包含身份初始化、文件分类、字段映射、修正策略、扩展安装与归零确认
- **扩展自动化**：初始化环境时会优先搜索并安装必要的 VS Code 扩展，确保 Agent 能顺畅调用扩展能力
- **更快启动反馈**：首次启动时会先主动向用户自我介绍、说明正在检查什么、接下来还会检查什么，避免出现“卡住半天”让人误以为失效的情况

> **概括**：把原来需要手工逐条对照 Excel、反复复制粘贴、手动写报告的工作，变成"丢文件 → 说句话 → 确认 → 拿报告"的 4 步流水线。

### 01 — 表格扫描与结构化

**触发方式**：说 `"导入数据"` / `"上传数据"` / `"帮我导入"`

**自动做了哪些事**：

| 步骤 | 做什么 | 你看到什么 |
| ---- | ------ | ---------- |
| ① 选择导入方式 | 已放入 `用户上传数据库/`，或指定系统路径文件夹自动扫描 | VS Code Ask Questions 选项 |
| ② 确认导入清单 | 指定路径模式下，先展示候选 Excel 清单，防止缺失或过多导入 | 清单正确 / 重新输入路径 / 包含子文件夹 / 取消 |
| ③ 逐文件询问 | 对每个文件弹出对话框：依据文件？待核对文件？ | 4 个选项按钮，点一下就行 |
| ④ 分类导入 | 依据文件→`原数据/依据文件/`，待核对→`原数据/待处理数据/` | 进度提示 |
| ⑤ 结构化扫描 | 只读取每个 sheet 的字段名、列索引、行数、数据类型，不读取明细值 | "识别到字段结构和行列规模" |
| ⑥ 生成 JSON | 生成不含原始单元格值的`表格结构扫描.json` | 就绪，提示下一步 |

**为什么只生成结构 JSON？**

Excel 每次打开需要解析，表格结构每次都要重新理解（哪行是表头？哪列是什么字段？）。V2.0 只把结构转成 JSON：

- AI 只看到 sheet、字段名、列索引、行列数，不接触合同、人员、客户、金额明细
- 核对时由本地 Python 脚本直接读取源 Excel 并输出安全摘要
- 即便调用第三方 AI API，原始财务数据也不会进入模型上下文

### 02 — 数据核对与校验

**触发方式**：说 `"核对"` / `"核对Q1"` / `"我要核对"`

**三层渐进式匹配逻辑**：

```text
对于季度分表中的每一行：
  ① 按「合同编号」在汇总表中查找
     → 找不到 → C 类（人工复核：合同仅季度有，汇总表缺）
     → 找到了 → 进入 ②
  ② 检查「人员」是否一致（季度分表的"姓名" vs 汇总表的"销售人员"）
     → 不一致 → B 类（不管金额对不对，人员不对必须先确认）
     → 一致 → 进入 ③
  ③ 检查「客户」是否一致（季度分表的"客户单位" vs 汇总表的"客户编码"）
     → 不一致 → B 类
     → 一致 → 进入 ④
  ④ 比较「金额」
     → 一致 → ✅ OK（完全匹配，无需处理）
     → 汇总为空、季度有值 → A1 类（自动填充，以季度分表为准）
     → 差异 ≤ 1 元 → A2 类（自动修正，以分表为准）
     → 差异 > 1 元 → B 类（待确认后才能修正）
```

**五级差异分级（A1 / A2 / B / C / OK）**：

| 级别  | 含义                                | 例子                                    | Agent 怎么做                 |
| ----- | ----------------------------------- | --------------------------------------- | ---------------------------- |
| ✅ OK | 三层全一致，金额也对                | 合同、人员、客户、金额全匹配            | 不管                         |
| A1    | 汇总表这行是空的，但季度表有数据    | 季度表有 1000 元，汇总表空白            | **自动填充**           |
| A2    | 金额差一点点（≤1元），人员客户都对 | 季度表 1000，汇总表 1000.35             | **自动修正**           |
| B     | 人员/客户对不上，或金额差很大       | 季度表张三→汇总表是李四 / 差了 1000 元 | **等你确认后再改**     |
| C     | 汇总表根本没有这个合同              | 季度表有个合同汇总表里找不到            | **标记出来你手动判断** |

**6 步交互式引导**（你不需要记任何字段名或路径，Agent 全部从文件里自动提取并弹出选项）：

1. 选依据文件（弹出文件列表让你点）
2. 选匹配字段（自动提取表头，默认推荐"合同编号"）
3. 选目标文件（弹出待核对文件列表）
4. 选核对字段（自动提取表头，默认推荐对应季度金额列）
5. 字段映射确认（如果名称不一致，帮你配对）
6. 最终确认 → 一键执行

### 03 — 修正执行与填充（V2.1 单工作副本模型）

**一句话功能**：确认后的差异，安全地写入汇总表（先备份、再修改、再验证、同文件跨季度累加）。

**触发方式**：核对完成后自动询问修正策略，或说 `"填充"` / `"我要修正"`

**🆕 V2.1 单工作副本模型**：同一汇总表只有一个工作副本 `修正后/{name}_工作副本.xlsx`。Q1→Q2→Q3→Q4 依次累加修改同一文件，**不再**产生 `_v1`/`_v2`/`_最终版` 等多版本。每季度修正前创建季度快照 `snap_{Q}_*.xlsx`，出错时从快照恢复后重试（最多 3 次）。

**执行流程（6 步安全链）**：

```text
步骤 0: 🔁 幂等性检查 + 工作副本定位
  → 检查修正方案是否已执行过
  → 检查 current_working_copy 工作状态
  → 若当前季度已修正则警告并确认
  → 工作副本不存在 → 首次从原数据复制
  → 工作副本已存在 → 直接在原文件上累加（不复写）

步骤 1: 🛡️ 创建季度快照（V2.1）
  → 对当前工作副本创建 snap_{Q}_{timestamp}.xlsx
  → 首版快照 + 季度快照链统一存入 修改日志/备份/
  → 更新 工作状态.json 中的 snapshot_chain

步骤 2: 📋 解析修正方案
  → 按核对报告中的 A1/A2/B 逐条处理

步骤 3: ✏️ 执行修正
  → 打开工作副本 → 定位行 → 写入新值 → 记录 before/after

步骤 4: 🔄 反向交叉校验（V2.0）
  → verification_engine.py 源分表 → 修正后汇总表逐项验证
  → 校验通过后自动更新 quarters_applied

步骤 5: 📝 写日志 + 幂等记录
  → 写入操作日志（含执行人、审核人、操作编码）
  → 写入已执行方案.json（防止下次重复执行）
```

**三种修正策略**：

| 策略                | 适合场景                                |
| ------------------- | --------------------------------------- |
| 仅执行 A 类自动修正 | 最安全、推荐——只修 Agent 确定没问题的 |
| A 自动 + B 逐条确认 | A 类自动修，B 类逐条确认后再修          |
| 逐条确认后执行      | 对每条差异单独决定，最精细              |

## 四大核心创新

### 创新一：🛡️ 三重数据备份机制 — 数据永不丢失、AI 无法肆意篡改

```text
                ┌──────────────────────────┐
                │     三重数据堡垒          │
                │                          │
  L1 原始凭证   │  用户上传数据库/          │
                │  → 永不修改、永不删除     │
                │  → 仅重命名标记导入状态   │
                │         ↓ 导入时复制      │
  L2 工作基准   │  数据库/原数据/            │
                │  → 只读保护               │
                │  → 所有Skill禁止修改此层   │
                │         ↓ 首次修改前快照   │
  L3 回滚防线   │  修改日志/备份/            │
                │  → 首版原始快照（仅一份）  │
                │  → 季度快照链 snap_Q1~Q4  │
                │  → 误操作按季度回滚       │
                └──────────────────────────┘
```

---

### 创新二： 算力节省机制 — 让 AI 只做真正需要智能的事

AI（大语言模型）按 Token 计费，每多处理一个字就多花一分成本。本系统从三个维度节省算力：

| 节省维度                 | 机制                                               | 效果                                            |
| ------------------------ | -------------------------------------------------- | ----------------------------------------------- |
| **结构化识别**     | Excel → 结构 JSON（仅字段/行列元数据），核对由本地脚本读取 Excel | 不需要每次重新理解表格结构，且原始数据不进入 AI 上下文 |
| **知识库固化**     | 字段映射、别名、核对规则、报告模板全部本地存储     | 零联网搜索，零外部 API 调用，节省 100% 搜索成本 |
| **工具脚本预处理** | Python 脚本执行 Excel 读写、匹配、对比等确定性计算 | AI 只读 JSON 结果做判断，不逐单元格处理数据     |

### 创新三：临时脚本隔离机制 — 定制需求不污染通用工具

面对不同表格格式或临时核对需求时，Agent 会先判断能否用现有通用脚本和配置 JSON 解决。只有确实需要一次性探查、特殊清洗、特殊核对或特殊验证时，才会新增 Python 脚本，并统一放入 `工具脚本/临时脚本/`。

| 场景 | Agent 做法 |
| --- | --- |
| 只是路径、sheet、字段列索引或阈值变化 | 不新增脚本，只生成配置 JSON |
| 现有通用脚本可支持 | 直接调用现有脚本 |
| 当前项目的一次性特殊处理 | 新增到 `工具脚本/临时脚本/` |
| 多次复用且跨项目稳定 | 先临时验证，再建议提升为通用脚本 |

临时脚本必须以 `temp_` 开头，不得在文件名或输出中包含客户名、人员名、合同号、金额明细等敏感业务信息。归零时会清空临时脚本和运行产物，保留目录说明文件。

### 创新五：一键归零 — 清空运行数据和临时脚本（V2.1 增强）

当项目架构复杂、数据堆积过多或需要反复功能测试时，手动清理既费时又容易误删核心文件。归零功能一键清空所有运行数据，系统瞬间退回至刚配置完环境的状态：

```text
用户说"归零"
    │
    ▼
┌─────────────────────────────────────┐
│  🔴 将被清除                         │
│  ├─ 原始数据 / 结构化数据             │
│  ├─ 所有报告 / 处理后数据             │
│  ├─ 备份快照 / 操作日志               │
│  ├─ 工作状态 / 已执行方案             │
│  ├─ 用户身份信息                      │
│  ├─ 导入标记                         │
│  └─ 临时脚本运行产物                  │
│                                      │
│  🟢 将被保留                         │
│  ├─ Python 环境 (.venv)              │
│  ├─ VS Code 扩展                     │
│  ├─ 知识库配置 / 通用工具脚本         │
│  ├─ 临时脚本目录说明文件              │
│  └─ Agent / Skill 文件               │
└─────────────────────────────────────┘
    │
    ▼
  系统退回"刚配置完环境"状态
  说"你好"即可重新开始
```

| 场景                       | 传统方式               | 归零功能                     |

| 多次测试后堆积大量中间文件 | 逐层清理费时费力       | 自动完整清理，核心文件零影响 |

> 🔒 **安全机制**：归零前双重确认，绝不触碰 Python 环境、扩展、知识库和通用工具脚本；`工具脚本/临时脚本/` 下的临时脚本和运行产物会一并清除。

### 创新六：🔄 单工作副本版本链（V2.1）— 根治版本扩散

过去每次修正一个季度就产生一份新文件（`_已修正`/`_v2`/`_最终版`/`_重制版`），跨季度累计修改时极易混淆和丢失。V2.1 单工作副本模型从根本上解决此问题：

```text
修正前  → 修正后/
           ├── 汇总表_v1.xlsx        ❌ 旧方式：每季度一个文件
           ├── 汇总表_v2.xlsx
           ├── 汇总表_最终版.xlsx
           └── 汇总表_重制版.xlsx

V2.1 后 → 修正后/
           └── 汇总表_工作副本.xlsx  ✅ 唯一工作副本，跨季度累加

           修改日志/备份/
           ├── 汇总表_原始.xlsx      # 首版快照
           ├── snap_Q1_20260724.xlsx # Q1 修正前快照
           ├── snap_Q2_20260724.xlsx # Q2 修正前快照
           ├── snap_Q3_20260724.xlsx # Q3 修正前快照
           └── snap_Q4_20260724.xlsx # Q4 修正前快照

出错恢复:
  工作副本损坏 → 从 snap_Q{n}_*.xlsx 恢复 → 重试（最多3次）→ 仍失败则标记人工介入
```

**核心约束**：
- 禁止使用 `_v1`/`_v2`/`_最终版`/`_重制版` 等版本后缀，违规即中止
- `工作状态.json` 自动追踪 `current_working_copy`（路径/已修正季度/快照链）
- Agent 执行修正前必须检查 `quarters_applied`，若当前季度已修正则警告确认

Agent 先弹出导入方式选择 → 如果选择指定系统路径，会先弹出候选文件清单确认 → 逐个弹出对话框让你选"依据文件"还是"待核对" → 导入完成后自动扫描结构 → 显示 "识别到字段结构和行列规模"。

#### 第 3 步：说"核对Q1"

Agent 自动选中 Q1 分表 + 对应季度金额列 → 弹出字段选择 → 你点确认 → 几秒后核对报告生成。

#### 第 4 步：确认修正策略

Agent 展示差异统计 → 你选"仅执行 A 类自动修正"（推荐）→ 自动备份 → 自动修正 → 修正报告生成。

#### 第 5 步：重复 Q2/Q3/Q4，最后说"出报告"

四个季度都完成后，说"出报告"→ 年度汇总报告生成，包含全年全景分析。

## ❓ 常见问题

### Q：我需要手动提供哪些信息？

| 必须手动提供                  | Agent 自动完成              |
| ----------------------------- | --------------------------- |
| 你的姓名（执行人）            | ❌ 读取 Excel、识别字段结构 |
| 审核人姓名                    | ❌ 差异分级（A1/A2/B/C/OK） |
| 操作编码（选填）              | ❌ 备份 → 修正 → 验证     |
| 客户编码表（仅脱密功能）      | ❌ 生成报告、写操作日志     |
| 原始 Excel 文件               | ❌ 字段名映射、别名匹配     |
| 每个文件的性质（依据/待核对） | ❌ 断点续跑（记住做到哪了） |

### Q：Agent 能输出什么？

| 输出           | 在哪个文件夹                                                                    |
| -------------- | ------------------------------------------------------------------------------- |
| 季度核对报告   | `${project_root}/结果输出/核对结果/核对报告/QX核对报告_日期.md`               |
| 季度修正报告   | `${project_root}/结果输出/核对结果/处理后报告/QX处理后报告_日期.md`           |
| 年度汇总报告   | `${project_root}/结果输出/核对结果/最终汇总报告/回款提成核对报告_2025年度.md` |
| 修正后的汇总表 | `${project_root}/结果输出/处理后数据/修正后/{文件名}_工作副本.xlsx`（V2.1 唯一工作副本） |
| 脱密后的文件   | `${project_root}/结果输出/处理后数据/脱密后/`                                 |
| 操作审计日志   | `${project_root}/修改日志/操作日志.md`                                        |
| 结构化数据     | `${project_root}/数据库/结构化数据/`                                          |

### Q：改错了怎么办？

从 `${project_root}/修改日志/备份/` 恢复：首版原始快照可完整回滚，季度快照链 `snap_Q1~Q4` 可按季度精确恢复。Agent 每次修正前都会保存季度快照，你可随时回滚到任意季度修正前的状态。

### Q：如果我做到一半电脑关机了，需要重来吗？

不需要。Agent 有**断点续跑**功能。重新打开后会自动检测上次做到哪一步，并问你是否从断点继续。

### Q：同一个修正方案会不会被执行两次？

不会。Agent 有**幂等性保护**——修正方案通过 SHA256 哈希去重，已执行的方案会自动跳过。

### Q：万一 Excel 被其他程序（如 WPS）打开了怎么办？

Agent 会检测到文件被锁定，并提示你"请关闭 Excel 后重试"，不会强行操作。

### Q：可以同时处理多个年度的数据吗？

可以。说 `"新建项目空间：2026年度回款提成"`，Agent 会创建一个独立的工作区。不同年度的数据完全隔离，互不干扰。

### Q：我需要联网吗？

除了首次安装 Python 和依赖包，后续所有操作**完全离线运行**。你的财务数据始终在你自己的电脑上，不会上传到任何服务器。

### Q：汇总表的列名跟模板不一样怎么办？

Agent 在导入时会自动扫描表头字段，并弹出对话框让你确认字段映射关系。你也可以提前在 `知识库/字段映射库/` 的 JSON 文件里配置好不同季度的字段名列索引。

### Q：同一个合同在不同文件里编号写法不一样怎么办？

在 `知识库/别名库/合同别名.json` 里配置别名映射，例如 `"2303Q": ["2303Q/2303H"]`，Agent 会自动把别名统一为标准名称后再匹配。

### Q：想做功能测试，但不想弄乱现有数据怎么办？

使用**归零功能**。说 `"归零"` 即可：先检查项目空间是否存在最终汇总报告或处理后数据，若有则提醒先导出保存 → 双重确认后一键清空所有运行数据（报告、数据文件、身份信息、临时脚本运行产物），系统退回至刚配置完环境的状态。归零不会影响 Python 环境、VS Code 扩展、知识库配置和通用工具脚本。测试完成后可以再次归零，重新开始正式使用。

### Q：归零会不会把我的配置也删掉？

不会。归零**只清除运行数据**（原始数据、报告、备份、日志、身份信息、临时脚本运行产物），**核心配置全部保留**：核对规则、字段映射、别名库、报告模板、客户编码表、Python 环境、VS Code 扩展、通用工具脚本等均不受影响。
